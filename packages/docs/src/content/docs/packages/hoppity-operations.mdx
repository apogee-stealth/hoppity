Typed runtime broker operations for hoppity-contracts. Wires contract-based event, command, and RPC handlers into the broker and extends it with typed outbound methods (`publishEvent`, `sendCommand`, `request`, `cancelRequest`).

Contract objects are the only accepted API surface — no string fallback.

## Installation

```bash
npm install @apogeelabs/hoppity-operations
```

:::note
Requires `@apogeelabs/hoppity`, `@apogeelabs/hoppity-contracts`, and `rascal@^20.1.1` as peer dependencies.
:::

## Usage

```typescript
import hoppity from "@apogeelabs/hoppity";
import {
    withOperations,
    onEvent,
    onCommand,
    onRpc,
    type OperationsBroker,
} from "@apogeelabs/hoppity-operations";

const broker = (await hoppity
    .withTopology(topology)
    .use(
        withOperations({
            serviceName: "warehouse",
            instanceId: crypto.randomUUID(),
            handlers: [
                onEvent(Order.events.placed, async (content, context) => {
                    console.log("Order placed:", content.orderId);
                }),
                onCommand(Order.commands.fulfill, async (content, context) => {
                    await fulfillOrder(content.orderId, content.warehouseId);
                }),
                onRpc(Inventory.rpc.check, async (request, context) => {
                    return { available: true, quantity: 42 };
                }),
            ],
        })
    )
    .build()) as OperationsBroker;
```

## Handler Declarations

Use `onEvent`, `onCommand`, and `onRpc` to declare typed handlers:

```typescript
// Event handler — fire-and-forget
onEvent(Order.events.placed, async (content, context) => {
    // content is typed from the Zod schema
});

// Command handler — fire-and-forget
onCommand(Order.commands.fulfill, async (content, context) => {
    // content is typed from the Zod schema
});

// RPC handler — must return a response
onRpc(Inventory.rpc.check, async (request, context) => {
    // request is typed, return type is inferred from the response schema
    return { available: true, quantity: 42 };
});
```

Each handler receives a `HandlerContext` with access to the `broker` for publishing events or sending commands from within a handler.

## Outbound Methods

The middleware patches typed methods onto the broker:

```typescript
// Publish a typed event
await broker.publishEvent(Order.events.placed, {
    orderId: "123",
    total: 49.99,
});

// Send a typed command
await broker.sendCommand(Order.commands.fulfill, {
    orderId: "123",
    warehouseId: "west-1",
});

// Make a typed RPC call
const result = await broker.request(Inventory.rpc.check, {
    itemId: "SKU-42",
});

// Cancel an in-flight RPC request
broker.cancelRequest(correlationId);
```

All outbound methods validate against the contract's Zod schema — type errors are caught at compile time, validation errors at runtime.

## Configuration

`withOperations` accepts an `OperationsMiddlewareOptions` object:

| Option             | Type                   | Required | Default | Description                                  |
| ------------------ | ---------------------- | -------- | ------- | -------------------------------------------- |
| `serviceName`      | `string`               | Yes      | —       | Service name for queue/exchange naming       |
| `instanceId`       | `string`               | Yes      | —       | Unique instance ID for per-instance queues   |
| `handlers`         | `HandlerDeclaration[]` | Yes      | —       | Array of `onEvent`/`onCommand`/`onRpc` calls |
| `defaultTimeout`   | `number`               | No       | `30000` | Default RPC timeout (ms)                     |
| `validateInbound`  | `boolean`              | No       | `true`  | Validate inbound messages against schemas    |
| `validateOutbound` | `boolean`              | No       | `true`  | Validate outbound messages against schemas   |

## Relationship to hoppity-contracts

`hoppity-contracts` defines the _what_ — domain contracts, topology generation, schema validation. `hoppity-operations` handles the _how_ — runtime handler wiring and typed broker methods. They're designed to be used together:

```typescript
// contracts package: define the domain
const Order = defineDomain("order", { ... });
const topology = buildServiceTopology(baseConfig, "order-svc", (t) => { ... });

// operations package: wire it up
const broker = await hoppity
    .withTopology(topology)
    .use(withOperations({ serviceName: "order-svc", instanceId, handlers }))
    .build();
```

See the [bookstore example](/hoppity/examples/bookstore/) for a full multi-service demo.
