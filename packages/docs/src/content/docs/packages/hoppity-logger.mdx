---
title: hoppity-logger
description: Custom logger injection for the hoppity middleware pipeline.
---

Middleware for injecting a custom logger into the hoppity middleware pipeline. Replaces the default `ConsoleLogger` with any logger that implements the `Logger` interface.

:::tip
This should be the **first** middleware in the pipeline so downstream middleware uses your custom logger.
:::

## Installation

```bash
npm install @apogeelabs/hoppity-logger
```

:::note
Requires `@apogeelabs/hoppity` and `rascal@^20.1.1` as peer dependencies.
:::

## Usage

```typescript
import hoppity from "@apogeelabs/hoppity";
import { withCustomLogger } from "@apogeelabs/hoppity-logger";

const broker = await hoppity
    .service("my-service", {
        connection: { url: "amqp://localhost" },
        handlers: [
            /* your handlers */
        ],
    })
    .use(withCustomLogger({ logger: myPinoLogger }))
    .use(otherMiddleware()) // this middleware will use myPinoLogger
    .build();
```

## Logger Interface

Your logger must implement the `Logger` interface:

```typescript
interface Logger {
    silly(message: string, ...args: any[]): void;
    debug(message: string, ...args: any[]): void;
    info(message: string, ...args: any[]): void;
    warn(message: string, ...args: any[]): void;
    error(message: string, ...args: any[]): void;
    critical(message: string, ...args: any[]): void;
}
```

Any logging library (pino, winston, bunyan, etc.) can be adapted to this interface with a thin wrapper.

## Configuration

`withCustomLogger` accepts a `WithCustomLoggerOptions` object:

| Option   | Type     | Required | Description                                |
| -------- | -------- | -------- | ------------------------------------------ |
| `logger` | `Logger` | Yes      | Logger instance implementing the interface |

## How It Works

The middleware sets `context.logger` to your provided logger. Since `MiddlewareContext` is shared across all middleware in the pipeline, every subsequent middleware — and the hoppity builder itself — will use your logger for all output.
