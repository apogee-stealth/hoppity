Hoppity ships with four example projects in the `examples/` directory. Each demonstrates a different messaging pattern and middleware combination. All examples include Docker Compose files for RabbitMQ and work out of the box with default settings.

## Prerequisites

All examples require:

- Node.js 22+
- pnpm 9+
- Docker (for RabbitMQ)

```bash
# From the repo root
docker compose up -d    # Start RabbitMQ
pnpm install            # Install all workspace dependencies
```

## Basic Pub/Sub

**Location:** `examples/basic-pubsub`

The simplest hoppity example. One publisher sends messages to a topic exchange, one subscriber consumes them from a queue.

**Middleware used:**

- `withCustomLogger` — custom logger injection
- `withSubscriptions` — auto-wired subscription handlers

**Key concepts:**

- Separate publisher/subscriber topologies (each service declares only what it needs)
- Subscription name mapping (`on_event` in topology matches the key in `withSubscriptions()`)
- Middleware ordering (`withCustomLogger` first, `withSubscriptions` last)

```bash
cd examples/basic-pubsub
pnpm dev:both
```

## Delayed Publish

**Location:** `examples/delayed-publish`

Two-service example demonstrating TTL-based delayed message delivery. A scheduler publishes messages with a configurable delay, a processor consumes them after the delay expires.

**Middleware used:**

- `withCustomLogger` — custom logger injection
- `withBasicServiceComms` — custom middleware (inbound/outbound exchanges)
- `withDelayedPublish` — delayed publish with TTL-based wait/ready queue infrastructure
- `withSubscriptions` — auto-wired subscription handlers

**Key concepts:**

- Wait queue with TTL → dead-letter exchange → ready queue → re-publish pattern
- `broker.delayedPublish()` for scheduling messages with a delay
- Custom middleware example (`withBasicServiceComms`)

```bash
cd examples/delayed-publish
pnpm dev:both
```

## RPC

**Location:** `examples/rpc`

Two-service example demonstrating request/response messaging over RabbitMQ. An initiator sends periodic RPC calls, a handler processes them and returns responses.

**Middleware used:**

- `withCustomLogger` — custom logger injection
- `withBasicServiceComms` — custom middleware (inbound/outbound exchanges)
- `withRpcSupport` — RPC exchange, queues, bindings, and broker methods

**Key concepts:**

- `broker.request()` — Promise-based RPC calls with automatic correlation ID tracking
- `broker.addRpcListener()` — register handlers that return response values
- Per-instance exclusive queues (reply + inbound) using `instanceId`
- `BrokerWithExtensions` for TypeScript-safe broker type combining multiple middleware

```bash
cd examples/rpc
pnpm dev:both
```

## Bookstore

**Location:** `examples/bookstore`

A multi-service demo showing the full contract-driven development pattern with `hoppity-contracts` and `hoppity-operations`. Two services (order-service and catalog-service) communicate through events, commands, and RPC, with a runner that drives the demo flow.

**Middleware used:**

- `withCustomLogger` — custom logger injection
- `withOutboundExchange` — per-service fanout exchange for observability
- `withOperations` — typed handlers and broker methods from domain contracts

**Key concepts:**

- `defineDomain` — shared Zod-validated contracts as the source of truth
- `buildServiceTopology` — declarative service topology from contracts
- `withOperations` — typed `broker.publishEvent()`, `broker.sendCommand()`, `broker.request()`
- Outbound exchange tapping — observing a service's traffic without coupling
- Event vs. command vs. RPC — when to use each operation type

```bash
cd examples/bookstore
pnpm --filter @bookstore/runner start
```

## Pattern Comparison

| Pattern          | Example         | When to Use                                                      |
| ---------------- | --------------- | ---------------------------------------------------------------- |
| Pub/Sub          | basic-pubsub    | Simple event broadcasting, decoupled consumers                   |
| Delayed Messages | delayed-publish | Scheduled notifications, retry with backoff, deferred processing |
| RPC              | rpc             | Synchronous request/response between services, queries           |
| Contract-Driven  | bookstore       | Multiple services with shared domain contracts, typed messaging  |
